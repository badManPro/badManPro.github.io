#前端学习记录-element-ui/MessageBox

##1.MessageBox简介
message-box是ElementUI的一个弹框，样式如图

一般用于提醒、警告、确认、权限等操作，而实现这样一个效果ElememtUI有非常方便的MessageBox。
##2. 调用方法

<template>
  <el-button type="text" @click="open">点击打开 Message Box</el-button>
</template>

<script>
  export default {
    methods: {
      open() {
        this.$confirm('此操作将永久删除该文件, 是否继续?', '提示', {
          confirmButtonText: '确定',
          cancelButtonText: '取消',
          type: 'warning'
        }).then(() => {
          console.log("成功")        
          }).catch(() => {
          console.log("取消")
        });
      }
    }
  }
</script>

其中，需要给触发元素绑定方法，一般为click,
其次要在调用的方法中调用this.$confirm，

  • 需要传入提示文字(string)作为第一个参数
  • 需要传入弹窗标题文字(string)作为第二个参数
  • 需要传入options作为第三个参数

##3. options参数

options内部文档有很多参数，具体请查阅官方文档
这里只介绍常用参数

参数 说明 默认值
type 消息类型，用于显示图标 -
cancelButtonText 取消按钮的文本内容 取消
confirmButtonText 确定按钮的文本内容 确定
center 是否为居中布局 false
distinguishCancelAndClose 区分取消和X按钮 false

##4. 需求实现
工作过程中接到一个需求，弹窗中取消和X按钮的动作是不同的。
distinguishCancelAndClose
于是便用到了distinguishCancelAndClose参数，传入参数后用action获取不同的状态进行判断然后调用不同的动作

warning() {
    this.$confirm("此操作将永久删除该文件，是否继续", "提示", {
        distinguishCancelAndClose: true,
        confirmButtonText: '确定',
        cancelButtonText: '取消',
        type: 'warning'
    }).then(() => {
        this.$message({
            type: 'success',
            message:"删除成功",
        });
    }).catch(action => {
        if(action === 'cancel'){
            this.$message({
            type: 'info',
            message:'已取消'
            })
        }else if(action === 'close') {
            this.$message({
            type: 'info',
            message:'已经关闭'
            })
        }

    })
}
